# Activity: Run Monte Carlo Simulation

**Activity ID**: 31
**Order**: 7
**Phase**: Simulation
**Dependencies**: Predecessor: Activity 29 (Produce Rough Estimates)

## Description

Run Monte Carlo Simulation

## Guidance

## Purpose
Execute Monte Carlo simulation (10,000 iterations, triangular distributions) over the scenario-level PERT triplets to produce probabilistic delivery forecasts: P50, P80, P95 for both token budget and calendar duration. Populate the Monte Carlo tab and the EBS (Evidence-Based Scheduling) diagram.

## Prerequisites
- EST-05 complete (Rough Estimates tab with min/expected/max per scenario and per sprint)
- OR EST-06 complete for Level 2 (Detailed Estimates tab with work-package-level PERT triplets)
- Monte Carlo VBS script present in the Monte Carlo tab of ESTIMATION_TEMPLATE.xlsx

## Steps

### Step 1: Verify Simulation Inputs

Before running, confirm in the Monte Carlo tab:
- Each scenario row has: scenario ID, sprint, min tokens, expected tokens, max tokens
- Sprint dependency order is correct (Sprint N cannot start before Sprint N-1 ends)
- Combined risk multiplier from Setup tab is referenced (simulation applies it per-iteration)
- Daily throughput (tokens/day) is set in Setup tab

If using Level 2 data, aggregate work-package PERT triplets to scenario level first (sum of WP min → scenario min, etc.).

### Step 2: Run the VBS Simulation

In the Monte Carlo tab, run the VBS macro (Alt+F8 → RunMonteCarloSimulation):

The macro performs 10,000 iterations. In each iteration:
1. For every scenario, draw a random token sample from Triangular(min, expected, max)
2. Apply sprint dependency graph (sequential constraint unless scenarios are flagged as parallelizable)
3. Compute total tokens for the iteration
4. Compute total duration for the iteration (tokens / daily throughput, respecting sprint boundaries)
5. Record [total_tokens, total_duration] to results array

Output columns populated by the macro:
- Column A: iteration token totals (sorted ascending)
- Column B: iteration duration totals (sorted ascending)
- Column C: cumulative probability (0.0 → 1.0)

### Step 3: Read Simulation Results

From the sorted output, extract:

| Percentile | Meaning | Token Budget | Duration |
|---|---|---|---|
| P10 | Only 10% chance of finishing within this | ___ K | ___ days |
| P50 | Median — 50/50 chance | ___ K | ___ days |
| P80 | 80% confidence — recommended for planning | ___ K | ___ days |
| P95 | Conservative commitment | ___ K | ___ days |

**Recommended planning budget: P80** (validated by ACM SAC 2021 — Monte Carlo achieves MMRE 20% vs 134% for developer estimates; P80 absorbs most of that residual error).

### Step 4: Produce EBS Diagram

The EBS (Evidence-Based Scheduling, Spolsky) diagram shows two cumulative probability curves:

**Curve 1: Token Budget Certainty**
- X-axis: token budget (K tokens) from P10 to P95
- Y-axis: probability of completing within that budget (0% → 100%)
- Mark calibration points: P50, P80, P95

**Curve 2: Duration Certainty**
- X-axis: calendar date (from today to P95 completion date)
- Y-axis: probability of completing by that date (0% → 100%)
- Mark specific dates: earliest (P10), expected (P50), planning target (P80), conservative (P95)

Both curves are generated by the VBS macro and rendered as embedded charts in the Monte Carlo tab.

Prepare the stakeholder communication:

### Step 6: Document Simulation Snapshot

Save a snapshot of simulation parameters and results to `docs/plans/MC_SNAPSHOT_{DATE}.md`. This becomes the baseline against which sprint actuals are compared in EST-08.

Fields to record:
- Date of simulation
- Input: total scenarios, total SP, convergence phase
- ECF/TCF combined multiplier
- K-token calibration status per size tier
- P50/P80/P95 results (tokens + duration)
- Sprint plan summary

## Rules to Follow

### I. Simulation Requires Min ≠ Max
If any scenario has min = expected = max (a single-point estimate), expand it: min = expected × 0.7, max = expected × 1.5. A zero-variance scenario creates false precision in the simulation.

### II. P80 Is the Planning Commitment
Never commit to P50 for stakeholder planning. P50 means you have a 50% chance of missing. P80 is the minimum defensible planning target. P95 for high-stakes commitments.

### III. Simulate After Each Sprint Close
The simulation is not a one-time event. After EST-08 (Sprint Close), remaining scenarios are re-simulated with updated PERT distributions. The EBS chart gains an "actuals vs. forecast" line each sprint.

### IV. Token Cost Is Computable
Multiply P80 tokens by current API pricing to give stakeholders a dollar estimate. Always show the price per MTok used in the calculation so it can be updated as pricing changes.
